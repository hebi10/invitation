# codex.md

이 파일은 AGENTS.md를 보완하는 Codex 전용 지침이다.
공통 규칙은 반드시 AGENTS.md를 따른다.
충돌 시 AGENTS.md를 우선한다.

## Codex 환경 제약
- Codex는 샌드박스 환경에서 실행되므로 네트워크 접근, 브라우저 실행, Firebase 실시간 연동이 제한될 수 있다.
- Firebase/외부 API 연동 코드는 로직만 수정하고, 실제 연결 테스트는 수동 확인 항목으로 표시한다.
- 검증은 lint/type/build까지를 기본 범위로 하고, 실행 불가능한 검증은 이유를 명시한다.

## Codex 전용 규칙
- 작업 전 먼저 관련 파일과 영향 범위를 파악한다.
- 요청 범위 밖 파일은 함부로 수정하지 않는다.
- 여러 파일을 수정해야 하면 먼저 수정 계획을 짧게 정리한다.
- 구조 변경이 필요한 경우 즉시 구현하지 말고 현재 구조 기준의 최소 수정안부터 검토한다.
- 서버 컴포넌트와 클라이언트 컴포넌트 경계를 깨지 않는다.

## 수정 전 체크리스트
- Firebase 관련 수정 시
  1. 클라이언트 코드인지 서버 전용 코드인지 확인
  2. `NEXT_PUBLIC_` 접두사 여부로 환경 변수 노출 위험 점검
  3. 이미지 업로드라면 관리자 경로(직접 업로드)인지 고객 경로(API 경유)인지 확인
- API 수정 시
  1. 요청 body
  2. 응답 shape
  3. 상태 코드
  4. 인증/권한 영향
- 폼/편집기 수정 시
  1. 초기값
  2. 검증
  3. 저장 중 상태
  4. 에러 메시지
  5. 성공 후 흐름
