# CLAUDE.md

이 파일은 AGENTS.md를 보완하는 Claude 전용 지침이다.
공통 규칙은 반드시 AGENTS.md를 따른다.
충돌 시 AGENTS.md를 우선한다.

## Claude 도구 활용 원칙
- 여러 파일을 수정해야 할 때는 영향 범위를 먼저 나열하고, 승인 후 진행한다.
- 큰 파일은 전체 재작성보다 부분 편집(Edit)을 우선한다.
- 파일 탐색 시 Glob/Grep 도구를 적극 활용하고, 추측으로 경로를 만들지 않는다.
- 수정 전 반드시 대상 파일을 Read로 먼저 확인한다.

## 응답 스타일
- 결론과 수정 방향을 먼저 제시하고, 배경 설명은 뒤에 둔다.
- 불필요한 배경 설명, 추상적 설계론, 과한 리팩터링 제안은 줄인다.
- "추천 구조"를 말할 때는 반드시 현재 구조 대비 장단점을 함께 적는다.
- 파일 분리가 필요하면 왜 필요한지와 분리 기준을 명확히 적는다.
- 완료 보고 시 "무엇을 바꿨는지"와 "무엇이 유지되는지"를 함께 적는다.

  1. 수정안 (바로 적용 가능한 코드)
  2. 영향 범위
  3. 검증 결과
- UI 피드백 요청
  1. 구조
  2. UX
  3. 접근성
  4. 유지보수성
  5. 우선 수정 항목
- 코드 리뷰 요청
  1. 현재 문제
  2. 위험도
  3. 수정 방향
  4. 최소 수정안
- 버그 수정 요청
  1. 원인 분석
  2. 수정안
  3. 재발 방지 포인트
