# 고객 이용권 지갑 기획

## 목적
- 모바일 결제, 웹 결제, 관리자 지급을 같은 기준으로 기록한다.
- PC에서는 주로 관리자가 제작권/운영 티켓을 지급하고, 고객은 보유 제작권으로 새 청첩장을 만들 수 있게 한다.
- 기존 이벤트별 티켓 잔액은 유지하되, 계정 단위 지급/소비 근거는 지갑 원장에 남긴다.

## 저장 기준
- `customerWallets/{uid}`는 Firebase Auth 고객 UID 기준 계정 지갑이다.
- `customerWallets/{uid}/ledger/{ledgerId}`는 지급, 구매, 소비, 이벤트 배정 이력을 기록한다.
- 제작권은 `standard`, `deluxe`, `premium` 등급별 수량으로 관리한다.
- 운영 티켓은 계정 단위 잔액으로 관리하고, 기존 `events/{eventId}.stats.ticketBalance`는 이벤트 귀속 티켓 잔액으로 유지한다.

## 현재 구현
- 관리자 고객 계정 탭에서 제작권과 운영 티켓을 지급할 수 있고, 현재 보유 수량과 최근 이용권 이력을 확인할 수 있다.
- 관리자 고객 계정 탭은 5개 단위 페이징과 검색으로 계정을 탐색한다.
- 고객 지갑 조회 API는 `/api/customer/wallet`이다.
- 고객 청첩장 생성 API는 `/api/customer/events` `POST`이며, 제작권 1개를 차감한 뒤 초안을 만들고 고객 소유권을 연결한다. 중간 실패 시 부분 생성된 초안을 정리하고 제작권을 환불한다.
- `/my-invitations/create`는 고객이 보유 제작권으로 한글 이름과 영문 이름을 입력해 새 청첩장 초안을 만들 수 있는 화면이다.
- 고객 직접 생성 화면의 청첩장 주소는 기존 page-wizard와 같이 신랑/신부 영문 이름을 기준으로 자동 생성한다.
- `/my-invitations`의 새 이벤트 만들기는 보유 제작권이 있을 때 고객 생성 화면으로 이동하고, 제작권이 없으면 관리자 문의 안내를 표시한다.
- `/my-invitations`의 연결된 이벤트 카드에서는 로그인 계정 소유권을 서버에서 확인한 뒤 팝업으로 방명록을 조회하고 삭제 예정 처리할 수 있다.
- 모바일 티켓팩 결제는 기존처럼 대상 청첩장에 바로 적립하고, 대상 이벤트가 고객 계정에 연결되어 있으면 고객 지갑 원장에 구매/배정 이력을 함께 남긴다.
- 결제 없는 모바일 직접 초안 생성 API는 `MOBILE_DRAFT_CREATION_ENABLED=true`일 때만 허용한다.

## 후속 작업
- 모바일 생성 결제는 Firebase 고객 UID를 RevenueCat `appUserId`로 사용하고, 생성 완료 시 이벤트 `ownerUid`를 연결한다. 따라서 모바일에서 만든 청첩장은 PC `/my-invitations`에서도 바로 조회된다.
- 모바일 미사용 제작권을 별도 지갑 잔액으로 판매하려면 RevenueCat 구매 원장을 `customerWallets/{uid}` 제작권 credit/debit과 연결하는 추가 설계가 필요하다.
- 이벤트별 티켓 사용 내역을 지갑 원장과 더 촘촘히 연결하려면 기존 `events.stats.ticketBalance`에 대한 마이그레이션 원장을 남긴다.
